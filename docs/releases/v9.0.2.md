# v9.0.2 — MCP OAuth Redirect URI Recovery

**Advanced Zebrunner MCP Server** patch release. Fixes `invalid_request: Unregistered redirect_uri` when connecting from **Cursor**, **Claude Code**, or after redeploy with stale OAuth client records.

---

## Highlights (v9.0.2)

### Fixed

- **Cursor native MCP OAuth** — recovered `mcp_*` clients now include `cursor://anysphere.cursor-mcp/oauth/callback`.
- **Claude Code** — loopback `/callback` redirect URIs added (Claude Desktop + `mcp-remote` continue to use `/oauth/callback` with RFC 8252 port matching).
- **Stale DCR on disk** — persisted clients saved with loopback-only `redirect_uris` after an earlier cold start are merged with the full canonical list on the next `getClient` call.

### Changed

- Optional env **`OAUTH_RECOVERED_REDIRECT_URIS`** — comma-separated extra redirect URIs for recovered clients without a code change.
- **`GET /health`** — reports `mcpServerUrl`, token/OAuth flow storage paths, on-disk counts, and `recoveredRedirectUriCount` for ops debugging. Pre-9.0.2 fields unchanged; HTTP 200 and `"status":"ok"` preserved for Docker/K8s probes.

### Ops notes

- No new required env vars beyond existing **`MCP_SERVER_URL`** and **`TOKEN_STORE_PATH`** volume (OAuth flow store under `dirname(TOKEN_STORE_PATH)/oauth-flow`).
- Redeploy the image, then reconnect MCP clients (credentials in `tokens.enc` are preserved).
