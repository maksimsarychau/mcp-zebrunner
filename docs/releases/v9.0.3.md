### Also included (9.0.2 + 9.0.1)

- **Cursor / Claude OAuth** — fixes `Unregistered redirect_uri`; merges stale `mcp_*` redirect URIs on disk

## Full documentation (repo)

**Advanced Zebrunner MCP Server** patch release: OAuth/security hardening, ops health visibility, and test reliability — on top of **9.0.2** redirect-uri recovery and **9.0.1** shared OAuth flow store for multi-replica HTTP.

### Important: tool names use the `adv_` prefix (v9.0.0+)

If you upgrade from **8.x** (or any pre-9.0 build), **every tool is registered under `adv_<name>`**. Legacy short names are **not** registered unless you opt in.

| Before (8.x) | Now (9.0.x) |
|--------------|-------------|
| `create_test_case` | `adv_create_test_case` |
| `list_test_runs` | `adv_list_test_runs` |
| `get_test_execution_history` | `adv_get_test_execution_history` |
| `detailed_analyze_launch_failures` | `adv_detailed_analyze_launch_failures` |
| … all ~61 tools | `adv_<same_name>` |

**Why:** Run this server **next to** Zebrunner's official hosted MCP (beta) at `https://{workspace}.zebrunner.com/api/mcp` without the 8 historical name collisions (`create_test_case`, `list_test_runs`, etc.).

**Migration options:**

1. **Recommended** — Update prompts, Cursor rules, and scripts to `adv_<name>`.
2. **Temporary** — Until migrated, set:

   ```bash
   ZEBRUNNER_REGISTER_LEGACY_ALIASES=true
   ```

   Legacy names work as deprecated aliases; metrics and descriptions still point at `adv_*`.

**Dual-MCP helpers:** resource `zebrunner://mcp-routing`, `about_mcp_tools` with `mode='routing'`, and descriptions prefixed with `[Advanced Zebrunner MCP]`. See [OFFICIAL_MCP_PARITY.md](../OFFICIAL_MCP_PARITY.md).

### Highlights (9.0.3)

#### Fixed

- **OAuth / HTTP security** — stricter query-param handling on auth routes; bearer token payload validation; safer OAuth flow store keys and file permissions.
- **Redirect URI env** — `OAUTH_RECOVERED_REDIRECT_URIS` entries are validated (loopback, `cursor://`, safe custom schemes); remote URLs are ignored.
- **E2E / local dev** — reliable server startup in tests; `MCP_SKIP_INTEGRITY_CHECK` documented for dev/E2E only (do **not** use in production).

#### Fixed (from 9.0.2 — reconnect MCP after deploy)

- OAuth **`Unregistered redirect_uri`** for **Cursor** (`cursor://anysphere.cursor-mcp/oauth/callback`), **Claude Code** (loopback `/callback`), and **Claude Desktop / mcp-remote** (loopback `/oauth/callback`).
- **Stale `mcp_*` clients on disk** — canonical redirect URIs merged on `getClient` after redeploy.

#### Fixed (from 9.0.1 — Kubernetes / multi-replica)

- **OAuth state across pods** — pending auth, issued codes, and DCR clients persisted under `dirname(TOKEN_STORE_PATH)/oauth-flow` (same volume as `tokens.enc`), fixing `Invalid or expired state` when login hits a different replica than authorize.

#### Changed

- **`GET /health`** — optional `storage` block (token path, OAuth flow directory, file counts, `recoveredRedirectUriCount`, `storage.health: ok|degraded`). **Probes unchanged:** HTTP **200**, top-level `"status":"ok"`.

### Upgrade

#### Docker (HTTP / selfauth)

```bash
docker pull msarychau/mcp-zebrunner:9.0.3
# redeploy your stack (compose, K8s rollout, etc.)
```

Redeploy with the same env as 9.0.2 (`MCP_SERVER_URL`, `TOKEN_STORE_PATH`, volume mount). Reconnect MCP clients after deploy.

#### Local STDIO

```bash
git fetch --tags && git checkout v9.0.3
npm ci && npm run build
```

Update MCP config to run `node /path/to/mcp-zebrunner/dist/index.js` (see [README.md](../../README.md) Dual-MCP example). Restart Cursor/Claude after the build.

For local dev only, if integrity check fails after editing source: `MCP_SKIP_INTEGRITY_CHECK=true` in env (not for production).

#### 8.x → 9.0.3

1. Rename tools in prompts → `adv_<name>` (or set `ZEBRUNNER_REGISTER_LEGACY_ALIASES=true`).
2. Deploy **9.0.3** image (Docker) or build from tag `v9.0.3` (STDIO); HTTP selfauth needs shared `/data` for `TOKEN_STORE_PATH`.
3. Reconnect Cursor/Claude; verify `/health` shows `storage.recoveredRedirectUriCount` ≥ 7.

**HTTP / selfauth (typical prod):**

- Keep `MCP_AUTH_MODE=selfauth`, `MCP_SERVER_URL`, `TOKEN_STORE_PATH`, and the `/data` volume.
- Optional: `OAUTH_RECOVERED_REDIRECT_URIS` only for **extra** allowed redirect URIs (not required for Cursor).
