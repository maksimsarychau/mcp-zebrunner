# v9.0.3 — OAuth hardening, health diagnostics, E2E reliability

Patch release bundling redirect-uri recovery (from 9.0.2), ops health visibility, security hardening, and test fixes.

---

## Highlights

### Fixed

- OAuth **`Unregistered redirect_uri`** for Cursor / Claude Code (see v9.0.2).
- **CodeQL query-param** type confusion on `/auth/callback`, `/login`, and related routes.
- **Bearer auth** validation for Okta/selfauth HTTP modes.
- **E2E** no longer fails when integrity signature is out of date after local source edits.

### Changed

- **`GET /health`** — optional `storage` object (paths, file counts, `storage.health`); Docker/K8s probes unchanged (HTTP 200, `"status":"ok"`).

---

## Upgrade (9.0.2 → 9.0.3)

```bash
docker pull msarychau/mcp-zebrunner:9.0.3
```

Redeploy with the same env as 9.0.2 (`MCP_SERVER_URL`, `TOKEN_STORE_PATH`, volume mount). Reconnect MCP clients after deploy.
